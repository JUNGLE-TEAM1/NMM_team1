Current Rule

L3는 AI-safe evidence, L4는 metadata/template retrieval, L5는 candidate grounding이다. L6는 Silver draft, L7는 Gold draft, L8는 vector/semantic handoff draft다. L10은 Silver spec, L11은 Gold spec, L12는 compiler validation이다.

대용량/실시간/비정형 데이터라도 AI는 row/event data-plane에 들어가지 않는다. AI는 L3의 bounded evidence를 보고 L6/L7/L8 draft를 추천하고, 사용자 승인 L9와 deterministic compiler L10/L11/L12 뒤에만 M2 preview 실행으로 넘어간다.

Direct M Handoff

M1 -> M3

M1 gives source registration to L0 and user edits/approval decisions to L9.

M3 -> M2

M3 sends only L10/L11 preview-only specs. M2 owns Spark/runtime execution and returns L13/L14 evidence.

M3 -> M5

M3 sends L9 approval state and L16 catalog sync package. M5 stores workflow, artifact, and catalog state.

M3 -> M6

M3 sends L16 SQL context and semantic/vector handoff with L15 caveats and exposure rules.
