M3 Expanded Layer Contract

기존 L0-L6에서 과밀해진 L3/L4를 L0-L10으로 재분해한 새 계약이다. 핵심은 L2 이후에 AI input pack, AI recommendation draft, user decision, deterministic compiler, Silver preview, Gold preview를 각각 분리하는 것이다.

왜 변경 L0-L10 실행 handoff 기존 매핑 모델 위치 역할 경계 개선 후보 선택 L0-L10 설계 해설 선택 통합 계약 선택 통합 계약 v2 선택 통합 계약 v2.1 선택 통합 계약 v2.1.1

변경 이유

기존 L3는 evidence reducer, Silver 추천, Gold 추천, approval decision이 한 단계에 들어갔다. 그래서 AI가 본 입력과 AI가 만든 추천, 사용자가 수정해야 할 결정이 섞였다.

기존 L4는 Silver spec, Gold spec, compiler validation, Silver preview validation, Gold preview validation이 한 단계에 들어갔다. 이러면 deterministic spec 생성과 preview 결과, Silver 구조 품질과 Gold semantic readiness가 한 status처럼 보인다.

새 구조는 AI 판단 위치를 명확히 하고, Silver와 Gold를 각각 수정/승인할 수 있게 하며, 실행 가능 spec은 AI draft가 아니라 deterministic compiler 산출물로 고정한다.

새 L0-L10 흐름

L0Raw Preservation
From M1 source registration
To M3 L1 bronze envelope

원본 pointer, checksum, byte size, replay 기준을 보존한다.

L1Bronze Envelope
From M3 L0 raw manifest
To M3 L2 profile snapshot

parse 실패, offset, rescue lane을 잃지 않는 bronze sample/envelope를 만든다.

L2Profile and Schema Snapshot
From M3 L1 bronze samples
To M3 L3 AI input evidence pack

unknown source를 schema fingerprint, field profile, sketch로 축약한다.

L3AI Input Evidence Pack
From M3 L2 profile/schema/sketch
To M3 L4 AI model slot

L2 evidence를 AI가 볼 수 있는 bounded control-plane package로 줄인다.

L4AI Recommendation Draft
From M3 L3 AI input pack
To M1/M5 L5 edit and decision

Codex/model이 Silver policy draft와 Gold model draft를 만든다.

L5Decision and Edit Contract
From M1 edits + M5 approval state
To M3 L6 spec compiler

사용자가 Silver와 Gold 추천을 따로 수정/승인한다.

L6Deterministic Spec Compiler
From M3 L5 approved decisions
To M2 L7/L8 preview execution

승인된 정책을 M2 Spark가 실행 가능한 Silver/Gold spec으로 컴파일한다.

L7Silver Preview Validation
From M2 executes silver spec
To M2 L8 Gold preview + M3 L9 gate

Silver schema, row count, masking, quarantine 구조를 검증한다.

L8Gold Preview Readiness
From M2 executes gold spec over Silver
To M3 L9 readiness gate

Gold metric, grain, measure, caveat, owner review 상태를 검증한다.

L9Quality / Drift / Quarantine
From M2 preview metrics + M3 gate rules
To M5/M6 L10 handoff

processing, drift, replay, catalog safety, quarantine gate를 분리 평가한다.

L10Catalog / Semantic Handoff
From M3 L0-L9 refs and gate results
To M5 catalog + M6 query context

M5/M6가 저장하고 질의할 catalog, lineage, SQL context를 넘긴다.

M4는 현재 실제 handoff path에 넣지 않는다. 팀에서 M4를 구체적인 execution/result producer로 배정할 때만 추가한다.

AI 모델 위치

gpt-5.3-codex-spark는 Apache Spark 실행이 아니라 L4 AI Recommendation Draft에 들어가는 모델 슬롯이다. 모델은 L3/ai_recommendation_input_pack.json만 입력으로 보고, L4/silver_policy_recommendation_draft.json와 L4/gold_model_recommendation_draft.json를 만든다.

L6부터는 AI가 아니라 deterministic compiler와 M2 Spark 실행 계약이다. AI draft가 곧 실행 spec이 되면 안 된다.
