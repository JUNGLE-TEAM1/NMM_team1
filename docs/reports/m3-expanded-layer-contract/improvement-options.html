M3 L0-L10 Improvement Options

각 계층별 개선 후보 3개를 단독 선택 가능한 완성형 방식으로 정리했다. 후보는 장점, 단점, 우리 구조에 유리한 이유, 유리한 파일 형식, 유리한 데이터 규모를 각각 3줄로 제공한다.

이 보드는 후보를 합쳐서 쓰라는 용도가 아니라, 이미 필요한 조합을 후보 안에 포함한 상태에서 하나를 고르는 용도다.

확장 계층 계약으로 돌아가기
선택 원칙과 실제 module handoff

억지 개선은 제외했다. 지금 구조는 이미 충분히 세분화되어 있으므로, 후보는 계층을 더 늘리는 방식보다 각 계층의 schema, validation, handoff, UI decision state를 더 명확히 만드는 방향이다.

실제 module handoff는 M1 source registration -> M3 recommendation/spec contract -> M2 preview execution -> M3 gate -> M5 catalog storage -> M6 query context로 유지한다.
