Logical L0

Raw Source Unit Manifest

원본 데이터를 복제하거나 변형하지 않고, replay 가능한 source 단위와 object identity를 고정하는 시작 단계다.

code: l0_raw.pyphysical: l0owner: M3 contract

이 단계가 하는 일

  • source root 아래 file/object를 탐색하고 object id와 source_unit_id를 만든다.
  • checksum, size, modified_at, URI를 manifest에 기록한다.
  • raw payload는 copy/mutate하지 않는다는 정책을 artifact로 고정한다.

코드와 산출물

코드tools/m3_contract/l0_raw.py
주요 산출물object_stream_manifest.json, source_manifest.json, raw_replay_pointer.json, raw_manifest.json
다음 입력L1 Bronze sample과 L2 profile의 replay 기준

판단 기준

L0가 통과하려면 source_unit_id, object_id, checksum/size/URI, replay pointer가 사라지면 안 된다. raw copy나 raw mutation을 M3 책임처럼 쓰면 실패다.

함수/코드 블록별 상세 해설

build_l0() signaturesource path, output dir, source_id, run_id, checksum mode를 받는다. 이 함수 하나가 L0의 모든 artifact를 만든다.
  • source가 파일인지 디렉터리인지 확인한다.
  • 후속 L1/L2가 재사용할 files, objects, source_units를 반환한다.
iter_source_files() + file loopsource 아래 파일을 안정적으로 순회하고 각 파일에 object id와 source_unit_id를 붙인다.
  • 현재 lightweight core는 파일 하나를 object_batch source unit 하나로 본다.
  • stream window는 core schema에 자리를 남기되 빈 배열로 둔다.
file_fingerprint()URI, path, size, modified_at, checksum을 만든다. checksum은 none, prefix, full mode로 비용을 조절한다.
object_stream_manifest bodysource_units, objects, stream_windows, raw_policy를 담는다. 여기의 raw_policy가 M3는 원본을 복제/변형하지 않는다는 계약이다.
raw_replay_pointerM2나 검증 하네스가 원본을 다시 찾을 때 필요한 object URI, source_unit_id, replay contract를 제공한다.
