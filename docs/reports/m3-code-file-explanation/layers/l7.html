Gold Recommendation Draft

Silver 위에서 만들 수 있는 Gold model, product-health template, risk_score policy draft를 만든다. 데이터셋에 없는 metric은 확정하지 않는다.

code: l4_recommendation.pyphysical: l4draft: Gold

이 단계가 하는 일

  • dimension/measure 조합으로 generic Gold aggregate 후보를 만든다.
  • product-health 최소 schema와 metric template 후보를 만든다.
  • risk_score component와 weight는 source evidence에 맞춰 adaptive하게 추천한다.

코드와 산출물

코드tools/m3_contract/l4_recommendation.py
주요 산출물gold_model_recommendation_draft.json, product_health_gold_template_draft.json, risk_score_policy_recommendation_draft.json
다음 입력L9 Gold decision, L11 Gold spec compiler

판단 기준

risk_score를 전역 고정 공식으로 설명하면 안 된다. negative review, low rating, low conversion, late delivery 중 evidence가 있는 component만 사용한다.

함수/코드 블록별 상세 해설

_gold_model_recommendations()dimension과 measure field를 조합해 generic aggregate Gold model 후보를 만든다. 둘 다 없으면 record_count fallback을 만든다.
_product_health_gold_template()product key, rating, review, conversion, delivery, time field를 찾아 product-health metric template과 required output columns를 만든다.
_risk_score_policy_recommendation()source evidence에 따라 risk component, weight, formula, missing component, zero denominator policy를 추천한다.
_risk_score_components()negative review, low rating, low conversion, late delivery component를 evidence가 있을 때만 추가한다.
_recommended_component_weights()사용 가능한 component만으로 base weight를 재정규화한다. 데이터에 없는 component를 0점처럼 넣지 않는다.
_metric_template()metric id, formula template, required evidence, available fields, missing evidence, owner review flag, caveat를 표준 형태로 만든다.
