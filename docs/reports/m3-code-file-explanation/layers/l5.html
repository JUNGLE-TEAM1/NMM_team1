Candidate Grounding Gate

Gold/product-health/risk/vector 후보가 실제 source evidence를 갖췄는지 검사한다. 후보를 추천하는 것과 실행 가능한 모델로 확정하는 것을 분리한다.

code: l3_recommend.pyphysical: l3gate: source evidence

이 단계가 하는 일

  • product key, rating, review, conversion, delivery evidence 존재를 확인한다.
  • missing evidence와 blocking reason을 기록한다.
  • PII/query exposure, Spark preview, user approval 필요 여부를 분리한다.

코드와 산출물

코드tools/m3_contract/l3_recommend.py
주요 산출물candidate_grounding_report.json, domain signal summaries
다음 입력L6 Silver draft, L7 Gold draft

판단 기준

데이터에 없는 지표를 Gold schema에 억지로 넣으면 안 된다. missing component는 risk_score에서 제외하거나 사용자 review로 넘겨야 한다.

함수/코드 블록별 상세 해설

_candidate_grounding_report()retrieved Gold template 후보가 실제 source field evidence를 갖췄는지 검사한다. exact name/type compatibility, denominator, PII exposure, Spark preview, user approval check를 분리한다.
product-health classifier helpers_product_id_fields(), rating/review/conversion/delivery helper가 field token과 semantic hint를 이용해 후보 field를 찾는다.
_product_health_missing_evidence()product id, rating/review, conversion numerator/denominator, delivery signal 부족분을 label로 만든다.
_matched_field_ids()matched field group에서 field id set을 뽑아 PII/query exposure validation에 연결한다.
gate outcome근거가 충분하면 L7 Gold draft로 넘어가고, 부족하면 needs_source_evidence나 owner review caveat로 남는다.
