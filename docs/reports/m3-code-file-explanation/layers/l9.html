Logical L9

User Decision and Approval State

L6-L8 추천 draft를 사용자가 수정/승인/보류/거절한 상태로 고정한다. Gold는 approved일 때만 compile 가능하다.

code: l5_decision.pyphysical: l5owner: M1/M5 decision state

이 단계가 하는 일

  • Silver policy decision을 preview 가능 상태로 고정한다.
  • Gold decision은 not_requested, deferred, needs_owner_review, approved, rejected로 분리한다.
  • Gold-to-Gold, product-health, risk, vector handoff를 별도 승인 대상으로 남긴다.

코드와 산출물

코드tools/m3_contract/l5_decision.py
주요 산출물approval_state.json, silver_policy_decision.json, gold_policy_decision.json, gold_to_gold_policy_decision.json, recommendation_diff.json
다음 입력L10/L11 deterministic compiler

판단 기준

Gold가 deferred/not_requested여도 Silver preview는 진행될 수 있다. Gold readiness가 Silver readiness를 오염시키면 L9 설계 실패다.

함수/코드 블록별 상세 해설

build_l5()physical l5 builder지만 logical L9 decision layer를 만든다. L4 draft와 gold_decision CLI 값을 받아 approval state를 고정한다.
VALID_GOLD_DECISIONSGold 상태를 not_requested, deferred, needs_owner_review, approved, rejected로 제한한다.
silver_policy_decisionSilver draft ref, approval status, selected fields, decision_trace_id, review_id를 담는다. 현재 Silver는 preview를 위해 기본 승인된다.
gold_policy_decisionGold가 approved일 때만 selected_models를 채운다. 그 외에는 draft는 보존하지만 compile 대상은 비워 둔다.
approval_stateSilver compile allowed와 Gold compile allowed를 분리한다. product-health, risk score, vector handoff는 별도 승인이 필요하다고 남긴다.
_gold_decision_reason()Gold 상태별 사람이 읽는 reason을 만들어 UI와 report에서 decision 의미가 사라지지 않게 한다.
