Gold Preview Readiness Gate

Gold preview/readiness evidence를 Silver readiness와 분리해서 남긴다. Gold가 not_requested/deferred여도 그 상태를 명시적으로 artifact화한다.

code: l8_gold_preview.pyphysical: l8gate: Gold readiness

이 단계가 하는 일

  • Gold preview ref와 execution status를 만든다.
  • approved Gold model이면 metric definition draft를 펼친다.
  • semantic caveat와 readiness candidate를 만든다.

코드와 산출물

코드tools/m3_contract/l8_gold_preview.py
주요 산출물gold_preview_ref.json, metric_definition_draft.json, gold_readiness_input_report.json, semantic_caveat_report.json
다음 입력L15 gold readiness axis

판단 기준

Gold not_requested/deferred/rejected는 실패와 다르다. 이 상태가 Silver pass를 막으면 안 된다.

함수/코드 블록별 상세 해설

build_l8()physical l8 builder지만 logical L14 Gold preview readiness evidence를 만든다. L5 Gold decision과 L6 Gold spec 결과를 읽는다.
gold_preview_refapproved여도 M3는 실행하지 않는다. 승인 전이면 execution status를 not_requested_or_deferred로 둔다.
_metric_definitions()selected Gold model의 measures를 metric id/name/operation/field/grain/semantic status 형태로 펼친다.
_semantic_status_candidate()Gold decision status와 metric 상태를 L15가 읽을 수 있는 pass/warn/block/deferred/not_requested로 변환한다.
_gold_caveats()not requested, deferred, rejected, owner review 상태별 caveat를 만들어 M6/context와 report에 전달한다.
