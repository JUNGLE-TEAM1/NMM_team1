Logical L16

Catalog, Query, and Vector Handoff Package

M3 core의 마지막 단계다. M5가 저장할 catalog sync package와 M6가 사용할 SQL/query/vector context를 만든다.

code: l10_handoff.pyphysical: l10to: M5/M6

이 단계가 하는 일

  • catalog metadata draft, SQL context pack, field lineage를 만든다.
  • semantic catalog vector index template을 만든다.
  • artifact_reference_manifest와 exports package를 만든다.

코드와 산출물

코드tools/m3_contract/l10_handoff.py
주요 산출물catalog_sync_contract_package.json, sql_context_pack.json, semantic_catalog_vector_index_template.json, artifact_reference_manifest.json, exports/*
다음 입력M5 catalog storage, M6 SQL/AI/vector context

판단 기준

L16은 package를 만든다. 실제 catalog DB write, SQL serving, vector index build는 downstream 책임이다. M3가 운영 저장소를 직접 소유한다고 말하면 안 된다.

함수/코드 블록별 상세 해설

build_l10()physical l10 builder지만 logical L16 handoff package를 만든다. L9 gate summary와 L5/L6/L7/L8 artifact들을 묶는다.
_allowed_columns()L5 Silver fields 중 query forbidden field를 제외하고 M6 query-safe column 목록을 만든다.
_catalog_metadata()publish 가능 여부, layer status, quality axes, semantic refs, caveats, lineage/sql refs를 catalog draft로 만든다.
_sql_context()M6가 사용할 SQL context pack을 만든다. Silver table은 기본 포함하고, Gold metric이 준비된 경우에만 Gold table을 추가한다.
_semantic_catalog_vector_index_template()dataset card, product-health template doc, schema field docs, metric docs, filter key, retrieval policy, accuracy boundary를 만든다.
_catalog_sync_package()M5/M6가 필요한 artifact refs, version set, m6_context_status, Gold metric 보유 여부를 하나로 묶는다.
_artifact_reference_manifest()output directory 아래 JSON artifact를 읽어 artifact id/name/access/logical/physical layer, checksum, URI, byte size를 manifest로 만든다.
export_*() functionsexport_transform_spec, export_schema_definition, export_workflow_definition, export_catalog_metadata가 M2/M5/M6 소비용 export 파일을 만든다.
