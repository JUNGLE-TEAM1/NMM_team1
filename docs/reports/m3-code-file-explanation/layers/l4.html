Logical L4

Metadata and Template Retrieval

profile/schema/catalog evidence로 Gold template, product-health 후보, vector/search 후보를 찾는다. 검색은 후보 발견용이지 metric 값 정확성 증명이 아니다.

code: l3_recommend.pyphysical: l3type: retrieval helper

이 단계가 하는 일

  • schema profile document와 built-in Gold template document를 만든다.
  • column profile documents와 payload filter key 후보를 구성한다.
  • product-health template 후보를 field evidence 기반으로 점수화한다.

코드와 산출물

코드tools/m3_contract/l3_recommend.py
주요 산출물metadata_retrieval_index_plan.json, gold_template_candidate_retrieval.json
다음 입력L5 candidate grounding gate

판단 기준

VectorDB나 metadata search는 추천 정확도를 보조하지만, source evidence 없이 Gold를 확정하게 만들면 안 된다.

함수/코드 블록별 상세 해설

_metadata_retrieval_index_plan()schema profile document, built-in Gold template document, column profile documents를 만든다. 나중에 vector/search index로 넘길 수 있는 문서형 metadata다.
built-in gold_product_health_v1 templateproduct_id, rating, review, conversion, delivery 같은 semantic evidence가 있으면 product-health 후보로 검색될 수 있게 한다.
_gold_template_candidate_retrieval()product/rating/review/conversion/delivery evidence를 찾아 후보 점수와 missing evidence를 계산한다.
payload_filter_key / blocked inputVectorDB에 넣을 수 있는 metadata와 넣으면 안 되는 raw payload 성격을 분리한다.
정확성 boundary검색은 후보 발견을 돕지만 metric 값의 정확성을 증명하지 않는다. 정확성은 L5 grounding, L9 approval, L13-L15 evidence gate에서 다룬다.
