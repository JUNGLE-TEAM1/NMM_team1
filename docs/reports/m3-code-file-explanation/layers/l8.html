Vector and Semantic Handoff Draft

schema/profile/catalog metadata를 vector/search extension으로 넘길 후보 template을 만든다. M3 core가 embedding을 만들거나 vector index를 운영하지는 않는다.

code: l4_recommendation.pyphysical: l4draft: vector handoff

이 단계가 하는 일

  • text/entity/metadata 후보 field를 분리한다.
  • chunking, privacy, index build extension hook을 제안한다.
  • AI generation trace에 raw payload 미노출과 row-level AI call 0을 기록한다.

코드와 산출물

코드tools/m3_contract/l4_recommendation.py
주요 산출물vector_embedding_handoff_template.json, ai_generation_trace.json
다음 입력L9 vector approval, L16 semantic catalog vector index template

판단 기준

Vector handoff는 catalog/search 정확도 보조다. metric numeric correctness를 증명하는 장치로 설명하면 안 된다.

_vector_candidate_fields()review/text/entity/metadata로 쓸 수 있는 field를 분류한다. PII 후보는 기본적으로 제외한다.
ai_generation_traceAI slot, deterministic fallback 여부, raw payload 미노출, row-level AI call 0을 남긴다.
