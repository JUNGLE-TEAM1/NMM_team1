Logical L2

Profile and Data Shape Snapshot

unknown CSV/JSON/JSONL/Parquet/text source를 format detection, field summary, schema fingerprint, data shape contract로 요약한다.

code: l2_profile.pyphysical: l2input: L1 bounded sample

이 단계가 하는 일

  • format hint와 sample로 CSV/JSON/JSONL/text를 감지한다.
  • field별 type count, null ratio, examples, PII hint를 만든다.
  • 대용량/streaming claim을 위해 profile이 bounded evidence임을 명시한다.

코드와 산출물

코드tools/m3_contract/l2_profile.py
주요 산출물profile_snapshot.json, format_detection.json, schema_fingerprint.json, *_profile.json
다음 입력L3 AI-safe evidence reduction

판단 기준

모르는 구조여도 field/type/null/example 근거가 남아야 한다. Parquet처럼 core parser 한계를 넘는 source는 extension required로 정직하게 표시한다.

함수/코드 블록별 상세 해설

build_l2()L1 sample과 L0 file metadata를 받아 format detection, profile snapshot, schema fingerprint를 만든다.
detect_format()format hint가 있으면 따르고, auto면 확장자와 sample으로 JSONL/CSV/text를 판정한다. Parquet은 core parser가 직접 파싱하지 않고 extension required로 넘긴다.
_profile_jsonl() / _profile_json_document()JSONL row 또는 pretty JSON document를 flatten해서 field stats로 만든다. parse 실패는 error count로 남긴다.
_profile_csv()CSV dialect/header/width conflict를 추정하고 field별 type/example/null ratio를 만든다.
_data_shape_contract()structured/semi-structured/extension-required 판단, large source claim boundary, M2/M5/M6 handoff 전제를 기록한다.
schema_fingerprintfield summary와 parser stats를 stable hash로 묶어 같은 source 구조인지 비교할 수 있게 한다.
