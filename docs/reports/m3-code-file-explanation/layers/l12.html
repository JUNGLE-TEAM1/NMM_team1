Compiler Validation and Unsupported Action Gate

L10/L11 spec이 안전한 preview-only deterministic contract인지 검사한다. unsupported action과 runtime boundary 위반을 여기서 막는다.

code: l6_compiler.pyphysical: l6gate: compiler

이 단계가 하는 일

  • unsupported action, needs_review action을 block item으로 만든다.
  • per-row AI, generated code, unbounded collect, production write를 검사한다.
  • logical L0-L16 transform graph를 생성한다.

판단 기준

legacy window_id, production write, generated code, per-row AI가 남아 있으면 block이다. preview_only 외 write mode는 허용하지 않는다.

함수/코드 블록별 상세 해설

SUPPORTED_ACTIONScompiler가 deterministic preview spec으로 바꿀 수 있는 action vocabulary다. 여기에 없는 action은 unsupported로 잡힌다.
_unsupported_actions()needs_review 또는 지원하지 않는 action을 block item으로 만든다.
_compiler_validation()per-row AI, generated code, unbounded collect, preview-only write mode, unsupported action, legacy window_id를 검사한다.
_contains_legacy_window_id()dict/list를 재귀 탐색해 window_id 키가 남았는지 확인한다. 남아 있으면 validation block이다.
layered_transform_graphLOGICAL_LAYERS 전체를 노드로 만들고 인접 edge를 구성해 L0-L16 실행 의미 흐름을 artifact로 남긴다.
