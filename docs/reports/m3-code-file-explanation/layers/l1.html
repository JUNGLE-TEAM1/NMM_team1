Logical L1

Bronze Envelope and Rescue Lane

원본 전체를 materialize하지 않고 bounded sample을 Bronze envelope로 감싸며, parse 실패가 구조 분석 과정에서 사라지지 않게 rescue lane을 둔다.

code: l1_bronze.pyphysical: l1input: L0 manifest

이 단계가 하는 일

  • row/byte 상한 안에서 sample line을 읽는다.
  • object_id, line number, byte offset, raw hash를 Bronze record에 붙인다.
  • parse failure와 schema conflict를 잃지 않는 rescue lane 계약을 만든다.

코드와 산출물

코드tools/m3_contract/l1_bronze.py
주요 산출물bronze_envelope_samples.json, bronze_envelope_spec.json, rescue_lane.json, sample_records.jsonl
다음 입력L2 profile이 읽는 bounded sample

판단 기준

같은 record를 다시 찾을 수 있는 locator가 있어야 한다. parse 실패 record가 profile 전에 없어지면 L1 실패다.

함수/코드 블록별 상세 해설

build_l1()L0 manifest를 입력으로 받아 l1 폴더와 Bronze sample artifact를 만든다.
  • L0 objects를 path 기준 dict로 만든다.
  • _sample_bronze_records() 결과를 manifest/spec/rescue lane으로 감싼다.
_sample_bronze_records()파일을 binary line 단위로 읽으며 row limit과 byte limit을 동시에 지킨다.
  • line_number와 byte_offset을 replay locator로 남긴다.
  • raw bytes는 hash하고 preview는 replacement decode로 제한한다.
Bronze record 생성 블록record_id, source_unit_id, object_id, line/byte locator, payload preview, raw sha256, ingest time을 만든다. L2 profile은 이 bounded record만 본다.
rescue_lane manifestparse 불가 record와 schema conflict가 정상 sample과 분리되어도 추적 가능하도록 계약을 만든다.
bronze_envelope_specBronze record가 가져야 할 required columns와 parse status enum, replay locator rule을 고정한다.
