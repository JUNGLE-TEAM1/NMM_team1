Logical L15

Three-Axis Readiness Gate

processing quality, catalog safety, Gold readiness를 분리해 최종 readiness를 계산한다. 핵심은 Gold 문제가 Silver readiness를 오염시키지 않는 것이다.

code: l9_gate.pyphysical: l9gate: final readiness

이 단계가 하는 일

  • processing_quality_axis와 catalog_safety_axis로 Silver readiness를 계산한다.
  • gold_readiness_axis는 별도 상태로 계산한다.
  • m6_context_status, safe_to_run_silver, safe_to_run_gold를 만든다.

다음 입력L16 catalog/query/vector handoff

판단 기준

Silver는 processing quality와 catalog safety만으로 ready/block을 계산한다. Gold not_requested/deferred는 Gold context만 not_ready로 만들고 Silver ready를 막지 않는다.

함수/코드 블록별 상세 해설

build_l9()physical l9 builder지만 logical L15 three-axis readiness gate를 만든다. L7/L8 evidence를 읽어 axis status를 계산한다.
_processing_status()L7 Silver validation이 block이면 processing block, 아니면 pass로 둔다.
_catalog_status()PII handling과 query forbidden field를 보고 catalog safety를 pass/warn/block으로 만든다.
_gold_axis_status()L8 semantic candidate를 Gold readiness axis status로 변환한다. 모르는 값은 warn으로 낮춘다.
_m6_context()processing/catalog block이면 Silver blocked. Gold not_requested/deferred/rejected/block은 Gold context만 not_ready로 만든다.
gate_summarysafe_to_run_silver, safe_to_run_gold, m6_context_status, required caveats, axis refs를 최종 요약한다.
