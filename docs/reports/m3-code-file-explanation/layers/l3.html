Logical L3

AI-Safe Evidence Reduction

AI가 볼 수 있는 입력을 bounded, redacted, field-evidence 중심으로 줄인다. raw full payload와 row-level AI는 이 단계에서 차단된다.

code: l3_recommend.pyphysical: l3type: AI control-plane

이 단계가 하는 일

  • L2 field summary에서 AI 입력에 필요한 근거만 추린다.
  • PII 후보 examples를 redaction한다.
  • AI evidence budget과 raw payload 금지 규칙을 artifact에 적는다.

코드와 산출물

코드tools/m3_contract/l3_recommend.py
주요 산출물ai_recommendation_input_pack.json, field_evidence_reducer.json, redaction_map.json, policy_context_pack.json
다음 입력L4 template retrieval, L6-L8 draft recommendation

판단 기준

L3에 raw full text, unbounded sample, per-row AI call 전제가 들어가면 실패다. AI는 data-plane이 아니라 control-plane 추천 입력만 본다.

함수/코드 블록별 상세 해설

build_l3() field loopL2 fields를 순회하며 field_evidence와 redaction_map을 만든다. PII 후보 field는 example을 [REDACTED_PII]로 바꾼다.
ai_recommendation_input_packAI가 보는 유일한 입력 package다. profile ref, redaction map ref, field evidence, evidence budget, raw payload 금지 조건을 포함한다.
field_evidence_reducerL2 profile에서 L3 evidence로 줄이는 규칙을 기록한다. full raw payload drop, example cap, PII redaction이 핵심이다.
_unknown_data_recommendation_pack()unknown source에서 가능한 Silver cleaning, generic Gold aggregate, product-health, vector handoff 후보를 큰 범주로 정리한다.
_domain_signals()field token과 semantic hint로 product key, review, time series, conversion, delivery 신호를 감지한다.
_source_processing_contract()대용량/실시간/비정형 처리는 M3 data-plane이 아니라 M2/M6/extension boundary로 넘긴다는 계약을 남긴다.
