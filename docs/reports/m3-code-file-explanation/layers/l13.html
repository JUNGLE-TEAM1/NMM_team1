Silver Preview Evidence Gate

M3가 직접 Silver preview를 실행하지 않는다는 ref와, M2/local preview evidence를 검증할 기준을 만든다.

code: l7_silver_preview.pyphysical: l7gate: Silver evidence

이 단계가 하는 일

  • Silver preview execution owner를 M2로 명시한다.
  • compiler status, preview-only write mode, PII/query exposure를 검증한다.
  • PII exposure report와 Silver quality axis를 만든다.

코드와 산출물

주요 산출물silver_preview_ref.json, silver_preview_validation_result.json, pii_exposure_report.json, silver_quality_axis.json
다음 입력L15 processing/catalog axes

판단 기준

Silver readiness는 구조/PII/query exposure 중심이다. Gold가 보류되어도 L13 Silver evidence는 독립적으로 평가한다.

함수/코드 블록별 상세 해설

build_l7()physical l7 builder지만 logical L13 Silver preview evidence를 만든다. 실제 Spark 실행은 하지 않고 M2 execution ref와 validation artifact를 만든다.
silver_preview_refexecution owner는 M2, M3 execution status는 not_executed_by_m3, write mode는 preview_only로 둔다.
silver_preview_validation_resultcompiler pass 여부, preview-only write mode, PII exposure declared, query forbidden exclusion을 검증한다.

silver_quality_axisSilver preview validation과 PII report를 L15 processing/catalog axis가 읽을 수 있는 형태로 묶는다.
