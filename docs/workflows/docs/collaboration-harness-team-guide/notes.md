# 협업 하네스 팀 사용 가이드 노트

## 진행 메모

- 사용자 요청은 "하네스 도입자의 책임 설명"이 아니라 "팀원들이 하네스를 이해하고 올바르게 써서 도입자가 운영 책임을 혼자 떠안지 않게 하는 설명 문서"다.
- 참고 양식은 17주차 AskLake MVP 학습 문서의 목차/흐름도/체크리스트 구조와 backend-control 템플릿의 역할/위험/운영 표 구조를 반영했다.

## 결정

- 새 문서는 기존 초보자 설명 guide를 덮어쓰지 않고, 팀원이 바로 쓰는 요청 예시 중심의 별도 guide로 추가한다.
- 제품/API/schema 변경이 아니므로 Source of Truth 문서는 수정하지 않는다.

## 열린 질문

- 없음. 이후 팀 피드백이 생기면 새 Phase 또는 현재 문서 후속 보강으로 처리한다.
